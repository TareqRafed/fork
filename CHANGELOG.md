# Changelog

All notable changes to this project will be documented in this file.

## [0.1.0] - 2026-03-14

Initial release of Fork — container orchestration for firmware.

### Features

- **Recipe-based toolchain detection** — Fork scans your project and resolves the right container automatically from TOML recipe definitions, no manual configuration needed
- **Generic recipe format** — TOML recipes describe toolchain layers declaratively, making it easy to add support for new MCUs via a PR
- **Docker/Podman build isolation** — builds run inside a container matched to your workspace; no global toolchain installs required
- **`fork build`** — builds firmware for a target MCU using the detected toolchain
- **`fork run`** — runs arbitrary commands inside the resolved container

### Notes

- Flashing support is out of scope for now; Fork focuses purely on reproducible builds
- Only a handful of MCUs have recipes defined — contributions welcome in `recipes/`
